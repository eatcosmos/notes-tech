# 组队学习流程

**截止入群**：09月12日 19:30（星期六）

**开营仪式**：09月12日 21:00（星期六）

- 我们发现通过组队的方式进行学习能够坚持下来的学习者比单独学习坚持下来的学习者要多，学习效果要好，所以开营仪式的核心是辅助大家组队。

- 为了更方便的评选优秀团队和优秀队长，本次组队学习不允许跨群组队（如果发生，退群，选择退监督金，重新扫码加入新的学习群），每个队伍建议5~10人。

- 在开营仪式上：

- 队长或者希望成为队长的学习者先做自我介绍招募队员。

- 学习者自我介绍，大家相互认识。

- 队长和队员自由组队。

- 组队后，由队长在《队伍登记文档》中登记组队信息。

- 已组队的学习者按照【编号-队名-昵称】修改群名片，未组队的学习者按照【编号-昵称】修改群名片，学习者编号见《学习者编号文档》，以截止入群后提供的数据为准。

- 学习者编号文档：https://shimo.im/sheets/mtopHB1h9wcjuP0Q/MODOC/ 

- 组队信息登记文档：https://shimo.im/sheets/fFpjUABWwfk6u7kU/MODOC/ 



**学前准备**

- 了解学习输出方式的方式，选择适合自己的一种即可。

- 在CSDN、Github或简书平台上撰写技术博客。

- 在B站进行学习分享的直播。

- 了解通过问卷打卡的方法，在问卷的 “打卡链接” 处粘贴对应任务的技术博客或录播的网址即可，我们会查看大家的学习分享。

- 按照【编号-队名-昵称】修改群名片（无组队【编号-昵称】）。

- 未按照规则修改群昵称的学习者在第一次打卡之前会被清退。

- CSDN、Github、简书、B站等入坑指南自行搜索。



**学习过程**

- 根据Datawhale提供的学习材料以自学为主，遇到问题在群内讨论。

- 把遇到的问题和解决的方法记录在答疑手册中，利人利己，方便大家查阅。

- 答疑手册：https://shimo.im/docs/Bjaazvhols4ynJzo/ 

- 勘误汇总：[https://shimo.im/docs/](https://shimo.im/docs/47kgJ277vnh0edqV/)[47kgJ](https://shimo.im/docs/47kgJ277vnh0edqV/)[2](https://shimo.im/docs/47kgJ277vnh0edqV/)[77vnh0edqV/](https://shimo.im/docs/47kgJ277vnh0edqV/) 



**结营仪式：**根据各课程的完成情况来确定结营时间。

- 完成学习任务的学习者即可获得“结课证书”，并邀请大家进入Datawhale成员群。

- 教辅人员会为本次的优秀学习者、优秀队长和优秀团队颁发相应的证书。

- 我们会在Datawhale成员群开展一系列的活动，如读书会、自律计划、WhaleFace、WhalePaper等等，践行“和学习者一起成长”的愿景。

# 关于打卡



打卡内容介绍：打卡，即任选CSDN、Github、简书、B站等平台，将自己的学习体会，输出成**【学习分享】**后，将分享的链接，填至问卷的 “打卡链接” 中相应的位置。



打卡要求：

1、打卡形式不限，可撰写学习笔记，包含自己运行代码；可记录重要知识，记录自己总结的学习体会；可直播分享自己的学习过程。

2、希望能有自己的理解，不要照搬学习文档。
